# Add session field for tracking external automation sessions

Add an optional `session` field to issues for tracking which automation session (e.g., ccx) is working on the issue.

**Requirements:**
1. Add `session: Option<String>` field to Issue struct
2. Add `itack set-session <id> <session-name>` command to set/update the field
3. Add `itack clear-session <id>` command to clear the field (or `set-session <id> ""`)
4. Display session in `itack show` output when set
5. Optionally show session indicator in `itack list` / `itack board`

**Example usage:**
```bash
itack set-session 15 ccx-a1b2c3d4
itack show 15  # Shows "session: ccx-a1b2c3d4"
```

**Files to modify:**
- src/core/issue.rs - Add field to Issue struct
- src/commands/mod.rs - Register new subcommand
- src/commands/set_session.rs - New command implementation
- src/commands/show.rs - Display session in output
